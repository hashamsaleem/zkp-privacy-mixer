Why is this Zero-Knowledge?

In each round, the Prover generates a random masking key $r$ (a one-time pad) and commits to $H(r)$ and $H(PIN \oplus r)$. When challenged with 0, the Prover reveals $r$. When challenged with 1, the Prover reveals $PIN \oplus r$.

What the Verifier Learns: The Verifier only learns either $r$ or $PIN \oplus r$ (both of which look completely random). Since the Verifier never obtains both values in a single round, the secret PIN is never exposed, yielding Zero-Knowledge.

Merkle Privacy Mechanics

A smart contract stores deposited coins as leaves in a Merkle Tree. When withdrawing, a user provides a Merkle Path.

Anonymity Analysis: Since the verifier function verifyProof(leaf, proof, root) only processes the leaf value and sibling hashes to confirm root equality, it proves membership without revealing the index of the leaf. Thus, the transaction remains unlinkable within the anonymity set!
